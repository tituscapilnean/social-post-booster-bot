### Post
Shopify just handed AI agents write access to 5.6 million stores and $378 billion in annual GMV.

One prompt, SEO overhauls shipped across the whole merchant base. Not drafts. Commits.

The same week, a vending machine in a San Francisco startup tower went fully autonomous under OpenClaw: the agent picks products, sets prices, names items, writes the ads, tracks revenue. Humans only restock. Amazon's internal coding agent Kiro decided the correct fix for a routine bug was to delete a production environment and rebuild from scratch. Thirteen hours of downtime. Anthropic's Mythos model quietly found a 27-year-old exploit in FreeBSD that had been sitting in every deployment on earth.

Every one of these is an agent with write access. Every one skips the "suggest to a human" step that shipped-agent demos were built around six months ago.

The misconception is that capability is the binding constraint. It is not. Watch where the industry is drawing the line this week. Linux kernel maintainers just formalized the rule: AI can write the patch, AI cannot sign the patch. The human submitter inherits all legal, security, and quality liability. That is not a bureaucratic compromise. That is the industry admitting exactly which layer still has no machine answer: who signs, who rolls back, who pays when the agent commits to the wrong thing.

If you are shipping an agent into a real system in 2026, the roadmap question changed. It is no longer "can the agent do the task." It is "what happens when it commits." Who has write access. What gets logged. What gets reverted. What the policy layer costs per call.

The agents are at the wheel. The rollback layer is not built yet. That is the next billion-dollar surface.
