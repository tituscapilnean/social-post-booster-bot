### Post
Adobe and Blender now answer to the same prompt window. The interface layer underneath every desktop app for two decades just got pulled out from under it.

Anthropic shipped MCP connectors for Adobe Creative Cloud, Autodesk Fusion, Affinity by Canva, Blender, Splice, Ableton, SketchUp, and Resolume in a single release. Adobe confirmed Claude can orchestrate workflows across more than 50 Creative Cloud tools from one prompt. Bilawal Sidhu's demo of 2D-image-to-3D-Blender-scene through the connector hit the timeline the same day.

The thing collapsing underneath this release is the interface assumption baked into every desktop app since the early 2000s. The window you opened used to define what the work was. With MCP, the agent picks the window after you describe the outcome.

Two consequences for anyone shipping agentic products.

First, the value of a polished standalone UI compresses fast for any tool an MCP server can drive. If a connector lets the model batch-rename layers, debug a 3D scene, or render across Premiere and Firefly without you alt-tabbing, the user no longer needs to learn the toolbar. The toolbar becomes a fallback for cases the model gets wrong.

Second, the connector itself becomes the product surface. Whoever ships a stable, well-documented, well-permissioned MCP server in front of a domain owns the agent funnel into that domain. Splice, Affinity, and Autodesk all chose to expose theirs. The companies that did not are now competing against a connector someone else will ship for them.

For builders shipping into creative or engineering verticals, the moat is no longer the UI layer. It is the policy layer. Per-tool permissions, undo and rollback semantics, observability of what the agent actually changed in a 200-asset project, attribution when it gets the brief wrong.

The agent picked the tool. Who is auditing the result?
