### Post
The model layer commoditized this week. The AGI clause got removed in the same paragraph that gave OpenAI permission to ship on AWS.

Microsoft's exclusive license to OpenAI's IP became non-exclusive, the AGI capability trigger was deleted from the contract, and OpenAI signed a $38B AWS deal the same day. Microsoft retains a 27 percent stake and revenue share through 2030, capped. Codex and OpenAI models are landing on Amazon Bedrock alongside Bedrock Managed Agents. The model layer is now distributed across every major cloud a builder might run an agent workload on.

In parallel, Ming-Chi Kuo reported OpenAI is working with MediaTek, Qualcomm, and Luxshare on an agent-first phone targeting 2028. Camera, microphone, location, payments, contacts, biometrics. The richest context machine humans carry, redesigned around the agent rather than the app drawer.

Read the two moves as one strategy. OpenAI just secured distribution everywhere a workload can run, and is now building the device that captures what an agent needs to act on a user's behalf. The pattern matches Apple before iPhone more than it matches an LLM company.

For anyone shipping agents on someone else's model, the implication is concrete. Your model provider is no longer staying inside the API surface you currently consume. They are moving down to the silicon, sideways into the device, and upward into the workflows your product was supposed to own. The vertical-integration radius is the new competitive map.

The MCP gateway, the routing policy, the per-tool permission layer, the rollback hook. That is the surface a non-vertical-integrated player still controls. Build there, or build for a market about to be re-platformed around someone else's phone.

Where in your stack does an OpenAI-controlled device change your roadmap?
