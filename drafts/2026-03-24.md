### Post
JavaScript got mocked for being too far from the hardware. The C++ crowd had a point. They also lost the web.

What's happening with prompts and MCP feels identical. The engineers who've built careers close to the metal, tuning weights, optimizing inference, writing CUDA kernels, are watching an abstraction layer eat their leverage. They're right that prompts are far from the hardware. That's exactly what makes them powerful.

Look at what's already shipping. Paperclip is an agent orchestrator that manages a full team to run a business end-to-end, while you sleep. Stripe co-released the Machine Payment Protocol this month, reviving HTTP 402 "Payment Required" so agents can request, authorize, and settle transactions in milliseconds, without touching a human interface. Benedict Evans flagged this week that token budgets are now a standard line item in engineering compensation packages, because agentic coding burns hundreds of thousands of dollars in compute per developer per month.

The underlying dynamic is the same one that's played out in every programming era. Assembly gave way to C. C gave way to C++. Python and Java captured enterprise. JavaScript ate the front-end by abstracting away the browser's internals. In each transition, the layer that won was not the one closest to the hardware. It was the one closest to the problem.

The LLM is the new compiler. It does not execute instructions linearly. It resolves intent. Prompts are the new syntax. MCP is the new standard library. The engineers dismissing this are the C++ purists of 2026.

The builders who win won't necessarily understand transformer internals. They'll understand how to orchestrate systems that do.
