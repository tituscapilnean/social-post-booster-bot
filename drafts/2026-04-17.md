### Post
Same sticker price. Up to 35% more tokens per prompt. xhigh as the new default effort level. Opus 4.7 shipped today and the economics moved without anyone calling it a pricing change.

Wrote this post with 4.7. Hit my weekly Max cap by midmorning. Boris Cherny and Cat Wu shipped launch-day playbooks the same morning: front-load context, delegate like it's an engineer, let it verify itself. All correct. All silent on the real implication.

The model got better. The cost of running it at defaults got worse. And the industry's response is visible if you look one layer down from the benchmarks.

Hermes Agent crossed 72K stars this week running on a 3,575-character memory budget. Claude Code sessions can eat 129GB of RAM to hold state. VaultCrux is compiling 80K-token contexts down to 2K. Karpathy's LLM Wiki is replacing RAG in production pitches. Nate Jones published this morning that memory has replaced the model as the moat, and the X timeline is converging on the same read from Aaron Levie, Ramya Chinnadurai, Danny Dang, the Letta team.

The misconception is that Opus 4.7 is just a bigger engine. It is. The more useful read is that the frontier has stopped pricing itself as a model and started pricing itself as a working memory. The weekly caps are a signal, not an inconvenience. Anthropic cannot afford to let every prompt run at xhigh with 35% more tokens on a tokenizer they just shipped. The advisor pattern is an admission. Context compaction is an admission. The 3,575-char Hermes budget is a rebuttal.

If you're shipping an agent in 2026 and your memory layer is "whatever the provider does by default," you are subsidizing someone else's roadmap. The teams that win from here compress their own context, route their own judgment, and own the substrate the model runs on.

The model is getting cheaper to replace. The memory is getting more expensive to lose.
